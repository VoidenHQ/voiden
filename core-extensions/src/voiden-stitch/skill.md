## Extension: Voiden Stitch Runner

Runs multiple `.void` files in batch and aggregates assertion results into a single report. Insert with `/stitch`.

### stitch — Stitch Runner Block

An atom block that discovers `.void` files by glob pattern, runs each in order, and streams results to the Stitch Results panel.

```yaml
---
type: stitch
attrs:
  include: '["**/*.void"]'
  exclude: '[]'
  stopOnFailure: false
  delayBetweenFiles: 0
  isolateFiles: false
  environment: ""
---
```

#### Attributes

| Attr | Type | Default | Description |
|------|------|---------|-------------|
| `include` | JSON string array | `'["**/*.void"]'` | Glob patterns for files to include. Empty array = all files. |
| `exclude` | JSON string array | `'[]'` | Glob patterns for files to exclude. |
| `stopOnFailure` | boolean | `false` | Stop execution after the first file/assertion failure. |
| `delayBetweenFiles` | number (ms) | `0` | Pause between consecutive files (e.g. `500` for 500 ms). |
| `isolateFiles` | boolean | `false` | Reset runtime variables to pre-run snapshot before each file. |
| `environment` | string | `""` | Key of environment to use. Empty string = active environment. |

#### Glob Pattern Rules

- Only `.void` files are discovered and run — other file types are ignored regardless of the pattern
- Patterns are matched against **project-relative paths** — never use a leading `/` or `\`
- Always use **forward slashes** `/` as path separators, never backslashes `\`
- `**/*.void` — all `.void` files anywhere in the project
- `api/**/*.void` — all `.void` files under `api/` folder (NOT `/api/**/*.void`)
- `users/*.void` — `.void` files directly in `users/` folder
- `**/draft-*.void` — exclude draft files (put in `exclude`)
- The current file is always excluded from the run

**Wrong:** `\api/**/*.void` or `/api/**/*.void`
**Correct:** `api/**/*.void`

> **UIDs and labels in examples are illustrative only.** Always generate a fresh UUID v4 for every `uid`. Use a label that matches what the stitch block is for (e.g. `"Run Auth Suite"`, `"Smoke Tests"`).

#### Slash Command

```
/stitch
```

Inserts a stitch block at the cursor. Aliases: `/batch`, `/suite`, `/collection-runner`.

#### Keyboard Shortcuts (when focus is inside a stitch block)

| Key | Action |
|-----|--------|
| `⌘↵` / `Ctrl+↵` | Run stitch |
| `Escape` | Cancel running stitch |

#### Results Panel

Results open automatically in the right panel (or bottom panel, depending on response panel position). The Stitch Results sidebar shows:
- Per-file status: passed / failed / error / skipped
- Per-section request details (method, URL, status code, duration)
- Assertion breakdown: total / passed / failed with descriptions
- Overall summary bar (files and assertions counts)

#### How Multi-Section Files Are Handled

If a `.void` file contains `request-separator` nodes, each section is executed independently in order. Results are grouped per file → per section in the sidebar.

### Complete Example

```markdown
---
version: 0.20.1
generatedBy: Voiden app
note: This file is auto-generated by the Voiden app
generatedAt: 2025-01-15T10:30:00.000Z
---

# User API Suite

Run all user-related request files, stop on any failure.

```void
---
type: stitch
attrs:
  include: '["users/**/*.void"]'
  exclude: '["users/**/draft-*.void"]'
  # No leading slash — paths are project-relative with forward slashes only
  stopOnFailure: true
  delayBetweenFiles: 200
  isolateFiles: true
  environment: ""
---
```
